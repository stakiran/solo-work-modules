# ドラフト・ランチャー
**ドラフト・ランチャー** とは、相手にいきなり提出する前に、いったん下書きとして寝かせるという概念である。

## フォーマット例
以下は 2 件の投稿を扱っており、うち 1 件は 2 日寝かせた後に提出しており、もう 1 件は 5 日経ってもまだ提出していない。

```
2026/01/14 社内転職、～～の求人のカジュアル面談を申し込みたい
 正直今の職場から逃げたいだけで、興味があるわけじゃないので迷ってる
 2026/01/16 カジュアルだし、話を聞いてみるだけでもいいのかな？迷ってる……
 2026/01/18 話したいことややりたいことが何もなくて、完全に相手にリードしてもらう心持ちを持っていると自覚。さすがにダメじゃない？

✅2026/01/17 そろそろAさんに～～の苦情を言いたい
 2026/01/18 上司に相談したけど、やっぱり取り入ってもらえない
 2026/01/19 言うことにした！
 2026/01/20 やめてくれたので良かった

※なお今日は 2026/01/20 だとする。
```

## 構造
- ランチャー
    - 投稿
        - コメント
            - タイムスタンプ
            - 内容
        - コメント
        - ...
    - 投稿
    - ...

1-ランチャー N-投稿を持つ。

1-投稿 N-コメントを持つ。

1-コメントは 1-タイムスタンプ と 1-内容 を持つ。

## ドラフト・ランチャーのルール
- (Must) ランチャーに投稿を新規追加する際、またコメントを追加する際は必ずタイムスタンプをつける
- (Must) 投稿した日と、提出する日は必ず日をまたがせねばならない
- (Must) 提出した場合は、その旨をコメントとして書く。また ✅ の印も書く
- (Should) 内容は1日分あたり1行、または一言程度で端的に済ませるべきである
- (Must) 画像は使ってはならない。言語で書いて表現する
