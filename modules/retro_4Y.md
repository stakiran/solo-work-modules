# 振り返りの4Y
**振り返りの4Y** とは、KPT や YWT のような振り返りフレームワークの一種である。個人が使うものであり、以下から成る:

- Yatta やったこと
- Yokatta 良かったこと
- Yokunakatta 良くなかったこと
- Yaru やること

まず「やったこと」を洗い出していく。その中で「良かったこと」と「良くなかったこと」が思いつくだろうから、同様に書いていく。ある程度出てきたら、最後にそれらを踏まえて「やること」を書く。つまり典型的には

- 1: やったことを書く
- 2: 良かったことと良くなかったことを書く
- 3: やることを書く

の形で、やったことをベースとした振り返りを行い、最後には次にやることを出せるようになっている。

## フォーマット
好きにすればいいが、派閥が二つある。以下は Markdown 記法を例にする。

一つは並列派である:

```markdown
- やった
    - ...
- 良かった
    - ...
- 良くなかった
    - ...
- やる
    - ...
```

もう一つは構造派である:

```markdown
- (やったこと)
    - o (これに対する良かったこと)
    - o (これに対する良かったこと)
    - ...
    - x (これに対する良くなかったこと)
    - x (これに対する良くなかったこと)
    - ...
    - ! (これらから導いた、次にやること)
    - ! (これらから導いた、次にやること)
    - ...
```

並列派は横断を重視する。網羅性や論理性よりも、本質にフォーカスする。良さげな学びと「やること」が生まれたら、それでいいと考える。

構造派は論理を重視する。「やったこと」に対して深堀りしており、「やったこと」単位で改善していく営みに等しい。

## KPT や YWT との違い
- KPT は元々チーム用であり、書いた観点をもとに議論することが前提となっている。個人で使うには抽象的でやり、次の行動につなげにくい。特に個人の振り返りでは、Keep は書きづらいだろう。個人では問題や課題を扱った方がやりやすい。しかし、Problem という大げさなテーマでは（特に個人的な些細な事象を）書きづらい
    - 反面、4Y では「やったこと」や「良くなかったこと」を書くので、些細なことでも書きやすい。しかし「やったこと」がベースになっているので、やってないことからの憶測や妄想は書けないようになっており、バランスが良い
- YWT は個人向けであり、書きやすくもあるが、W は「わかったこと」であり、学びとして昇華しなければならない圧がある
    - 反面、4Y は「良かったこと」と「良くなかったこと」という形で印象や感想を書かせるので、書きやすい。必ずしも学びを得る必要もない。しかし、次の「やること」を導くには十分な情報であり、建設的である
