# List Kit
List Kit とはリストを管理するためのフレームワークである。どのように管理するかを決めるためのヒントを提示する。ここで管理とは設計、作成、運用、配布を含む。

List Kit はフレームワークや質問集といったツールから成る。

# List Kit を構成するツール集

## 3Lサイクル
List、Log、Lesson の略であり、1つのリストに対して、それを使用したときの記録（Log）と、ログをインプットにして行う考察とそこから得られた学び（Lesson）を付与する。

またリストの内容と様式は、Lesson の結果を受けて修正しても良い。むしろ積極的に修正した方が良い。3L サイクルはサイクルの名前のとおり、フィードバックループを回すものである。

例:

```markdown
## List (ここにタイトル)
- (リストの内容)
- ...

## Log
- (使ってみた感想、経過や結果)
- ...

## Lesson
- (ログに基づいた考察とそこから得られたこと)
- ...
```

またリストにはテンプレートとインスタンスがある。テンプレートは型であり、これに対して具体的な文脈と記載を行うとインスタンスになる。

たとえば「1行1タスクで書き、各行は単語を読点区切りで並べるタスクリスト」があるとして、wool.txt としてつくっている。Word Only and One Line の略だ。これはテンプレートであり、実際はここから wool_project1.txt、wool_travel1.txt、wool_資格勉強用.txt といったインスタンスがつくられるだろう。3L サイクルでは wool.txt の方でフォーマットを書いておき、インスタンスの方で具体的な感想や考察を書いていくと良い。wool.txt というテンプレートありきで、ここからインスタンスをつくるのだというモデルを忘れないようにせよ。

## 3Lサイクルの論点
- 3L は同じファイル内に書く？それとも分ける？
- Log と Lesson のタイムスタンプはどう書く？どう区切る？
- Log と Lesson の記入タイミングは？
- Log と Lesson の記入フォーマットは？

## リストを使うタイミングを設計する際のトリガーリスト
- As an input ですか？
    - 例: ～～を行う前のインプットとして読んでおこう
- As a tool ですか？
    - 例: ～～をするために使えるので使おう
- As an output ですか？
    - 例: 今から～～を残したいので、リストの形で残そう

## リストの使い方を設計する際のトリガーリスト:
- 順序性：上から順に見るかどうか
- 記入性：答えを書き込むかどうか
- 即時性：その場で行動させるかどうか（思考と記入は含めない。典型例は手順）
- 任意性：その項目をスルーしてもいいかどうか

## リストをどうやって使い始めるかを設計する際のトリガーリスト:
- そのリストの存在はいつ、どうやって思い出しますか？
- そのリストの存在を思い出させるための合図や仕組みはありますか？つくれますか？
- そのリストにはどうやってアクセスしますか？
- そのリストが記入を伴う場合、記入可能なファイルやページや用紙はどのようにしてつくりますか？

## リストのつくり方を考えるためのトリガーリスト:
手段:

- プレーンテキスト？: txt, Markdown etc
- リッチテキスト？: Word, PDF etc
- スライド？: Powerpoint, Keynote etc
- キャンバス？: Miro, Figma, Canva etc
- スプレッドシート？: Excel, Googleスプレッドシート etc
- ノートページ？: Google ドキュメント, Notion etc

観点:

- 保存のしやすさ
- アクセスのしやすさ
- 記入のしやすさ
- 共有のしやすさ
