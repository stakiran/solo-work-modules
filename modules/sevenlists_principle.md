# セブンリスト原則

## 人間向けのビジュアル

```
               Task    Span
     Trigger                  Question
            Check        Label
                   Fill
```

## 概要
**セブンリスト原則** とは、概念を道具化するのに有益なリスト（箇条書き）に関する原則であり、実用の観点では、主なつくりかたの方向性が 7 つあることを示したものである。

## セブンリスト解説

### 1: Task List
- これは何？
    - 指示を列挙します
- 読者への期待
    - 読者に従ってもらうことを期待します
- 例
    - SOP など手順書の類はここに含まれます

### 2: Span List
- これは何？
    - 期間単位（時間的な長さを持つ事柄）を並べます
    - 並べた期間単位の中に、さらに要素を置くこともあります（分類機能）
- 読者への期待
    - 期間の全体像と現状を知ってもらうことを期待します
- 例
    - ロードマップ、スケジュール、WBS

### 3: Trigger List
- これは何？
    - 発想の参考になりそうな事柄を雑多に並べます
- 読者への期待
    - 発想の助けになることを期待します
    - 各項目は使っても使わなくてもいいし、いいかげんに解釈しても良いです
- 例
    - GTD®のトリガーリスト、オズボーンのチェックリスト、SCAMPER法

### 4: Question List
- これは何？
    - 質問、特にオープン・クエスチョンを並べます
- 読者への期待
    - 質問に一つずつ答えていくことを期待します
    - 答えていくことで色々見えてきます、特に「読者自身の意思」と対峙し意思決定してもらうことを期待します
- 例
    - インタビューやサーベイ全般

### 5: Check List
- これは何？
    - Yes/No で答えられる事項を並べます
- 読者への期待
    - 指定事項が Yes なのか No なのかを確定させることを期待します
- 例
    - 割愛

### 6: Label List
- これは何？
    - 対象に付与する分類項目（ラベル）を並べます
    - 読者はこの中からラベルを選んで、対象に付与します
    - 1対象1ラベル（カテゴリー方式）、1対象nラベル（タグ方式）があります
- 読者への期待
    - 指定された分類項目を使って分類してもらうことを期待します
- 例
    - GitHub Issues のラベル、文書システムで用いるタグ

### 7: Fill List
- これは何？
    - 読者に埋めてもらう枠を並べます
    - いわゆるテンプレートです
- 読者への期待
    - 枠を埋めてもらうことを期待します
- 例
    - テンプレート全般
