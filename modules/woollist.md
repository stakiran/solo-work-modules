# WooL List
**Wool List** とは Word Only and One Line List の略であり、以下をすべて満たすタスクリストを指す。

- 1行1タスクで書く
- 各タスクは「単語」と「区切り」のみから構成する

これ以外のフォーマットは任意である。下記例では Markdown のリスト記法を使っているが何を使っても良い。

## 注意点
紛らわしいが、羊毛とは関係ないことに注意。

## Wool List の例
正しくない例を挙げる:

```
❌

- ミルクを買う
- 週次レビュー用のテンプレートを修正する
- Claude Code の Hook を試す
```

次に正しい例を挙げる。なお区切りとして読点を使うとする:

```
⭕

- ミルク
- 週次レビュー、テンプレート
- Claude Code、Hook
```

上記で思い出しづらいなら、以下のようにもう少し具体的にしても良い:

```
⭕

- ミルク、購入
- 週次レビュー、テンプレート、修正
- Claude Code、Hook、試行
```

また動作を強調したいなら語順を変えることもできる:

```
⭕

- 購入、ミルク
- 修正、週次レビュー、テンプレート
- 試行、Claude Code、Hook、
```

逆に想起に自信があるなら、大胆に情報を間引くこともできる:

```
⭕

- ミルク
- 週次レビュー
- Hook
```

なお、「単語」と「事前に定めた区切り」以外は使ってはならない。たとえば以下はどの行も NG である:

```
❌

- ミルク、買う
- 週次レビュー, テンプレ
- Claude Code の Hook 調べときたい
```

## トレードオフ
単語をたくさん書いたり、動作も書いたりすると思い出しやすいが、その分書く手間が増える。Wool List に頼りたい者はおそらく形骸化しがちな者であろうから、軽微な手間であっても形骸化につながるかもしれない。

逆に単語が少なすぎると、後で見たときに思い出せない可能性がある。あるいは思い出すのに時間がかかるなど、認知コストを消耗するかもしれない。いずれにせよ、スムーズに思い出せないタスクリストは通常形骸化する。

したがって、最適なバランスを探るべきである。
