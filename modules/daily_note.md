# デイリーノート
**デイリーノート** とは、日単位でノート（ページ）をつくって動線にすることである。

たとえば今日が 2026-02-12 なら 2026-02-12 というファイルやページをつくり、そこを動線とする――つまり日々のメモ、タスクリスト、検討などを何でも書き込むホームとする。日記と同様に思えるが、動線化している点と、タスクを始め実運用的な情報も書く点が違う。

日付が切り替わったらノートも切り替える。つまり一日の運用は「今日のノートをつくる」ことと「必要なら昨日のノートを読んで内容を転記する」ことから始まる。この際、コピペは良いが、自動生成はしない方がいい。なぜなら手作業による生成は、デイリーノートを整えるという面倒な作業の、最初の作業興奮を引き起こすために有益だからである。この点を理解せず、無闇に自動化してしまう者は、おそらくデイリーノートが定着しない。逆を言えば、すでに定着しているのならば自動化しても良い。

デイリーノートのフォーマットは自由だが、デイリータスクリスト（今日やるタスク）は見えるようにしておきたい。デイリータスクがすべて終わったら今日は終了、とみなせるためだ。

応用テクニックとして、未来日のデイリーノートを先につくっておくものがある。たとえば今日が 2026-02-12 だとして、2026-03-01 にやるべきことが出てきた場合、2026-03-01 のノートをつくっておいて、書いておくと良い。3月1日になったら思い出せるし、それまでは忘れていてもいい。頭の中に留める必要がなく、認知資源の節約になる。
